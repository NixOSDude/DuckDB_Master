// THE CODE WE ACTUALLY RAN

Plain SQL. Verifiable results.

Five queries. Real data from the NYC TLC open dataset. No proprietary runtime. Run it yourself.

Q1 — Total trip volume, 4 years
SELECT year, COUNT(*) AS total_trips, ROUND(COUNT(*) / 1e6, 2) AS millions FROM ( SELECT 2022 AS year FROM read_parquet('yellow/2022/*.parquet') UNION ALL SELECT 2023 FROM read_parquet('yellow/2023/*.parquet') UNION ALL SELECT 2024 FROM read_parquet('yellow/2024/*.parquet') UNION ALL SELECT 2025 FROM read_parquet('yellow/2025/*.parquet') ) GROUP BY year ORDER BY year;
Q3 — Monthly trend pivot (DuckDB-native PIVOT)
PIVOT ( SELECT year, MONTH(tpep_pickup_datetime) AS month, COUNT(*) AS trips FROM all_years GROUP BY year, month ) ON year USING SUM(trips) GROUP BY month ORDER BY month; -- Native PIVOT syntax — no Spark workarounds, no SparkSQL conditionals
